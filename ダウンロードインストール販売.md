# ブラウザゲームを「ダウンロード／インストール販売」する場合のメリット・デメリット

対象作品: 脳リフレクソ / Nebula Garden（脳リフレクソ2） / Fluid Words / Spacey Bloom  
いずれもブラウザ上で動く Canvas / Web Audio 系の癒し・リラックス系ゲーム。

---

## 1. 売り方は大きく3つ

| 形態 | 実態 | 作品との相性 |
|------|------|----------------|
| **A. Webのまま（URL課金／限定公開）** | Safariで開く | いちばん簡単。すでに近い |
| **B. ダウンロード販売（HTML/zip）** | 買ってファイルを入手 | 脳リフレクソ型。相性◎ |
| **C. インストール販売（App Store / Google Play など）** | ホーム画面のネイティブ風アプリ | Capacitor 等で包む必要。工数・審査あり |

「ダウンロードインストール販売」は多くの場合 **B か C** を指す。メリット・デメリットもここで分かれる。

---

## 2. ダウンロード販売（HTML / zip）のメリット

### 売り手側

- **開発をほぼ変えなくてよい**  
  Vanilla HTML/CSS/JS のまま。React 化やネイティブ書き直しが不要。
- **審査がない（または軽い）**  
  BOOTH・note・自社サイト・PayPal などなら、Apple の審査待ちがない。
- **手数料を抑えやすい**  
  App Store は売上の約15〜30%。ダウンロード販売は決済手数料程度に抑えられることが多い。
- **オフライン・所有感を売りにできる**  
  「一度買えばネットなしでも遊べる」「ファイルとして残る」は癒し系・繰り返し遊ぶ作品と相性がいい。
- **更新の主導権が自分にある**  
  ストア審査を待たずに、購入者へ新版 zip を配れる（運用ルールは自分で決める）。
- **既存の強みを活かせる**  
  単体 HTML（`Brain_Reflexo.html`）は iPhone の Files → Safari という導線と一致している。

### 買い手側

- アカウント必須のストアに縛られない（販売チャネルによる）
- 「アプリを入れる」心理的ハードルが低い人もいる（特にライトな癒し用途）
- 端末の容量・権限をあまり取らない印象になりやすい

---

## 3. ダウンロード販売（HTML / zip）のデメリット

### 技術・体験

- **iPhone が弱い**  
  zip 解凍 → Files →「Safariで開く」は、慣れていない人には離脱ポイント。
- **「インストールしたアプリ」感が出にくい**  
  ホーム画面追加はできても、アイコン・通知・背面ジェスチャなどはネイティブより劣る。
- **ブラウザ依存**  
  iOS 更新・Safari の仕様変更・消音スイッチで音が出ない、などで「壊れた」と思われやすい。サポート負担が増える。
- **コピーされやすい**  
  HTML/JS は中身が見える。完全な不正コピー防止は難しい（難読化は抑止程度）。
- **発見されにくい**  
  App Store の検索流入がない。集客は自分（SNS、広告、口コミ、BOOTH 等）頼み。

### ビジネス

- **価格を上げにくい**  
  「ブラウザのファイル」は、見た目の価値がアプリより低く見られがち（中身が良くても）。
- **返金・サポートが個人に集中しやすい**  
  「音が出ない」「開けない」は端末設定の問題でも、販売者に問い合わせが来る。
- **アップデート配布が面倒**  
  購入者リストへの再配布、新旧ファイルの混在、「どれが最新？」問題。

---

## 4. インストール販売（App Store 等）のメリット

### 売り手側

- **信頼と単価が上がりやすい**  
  「App Store のアプリ」は、zip より高く売りやすい・ギフトしやすい。
- **発見・導線が強い**  
  検索、カテゴリ、おすすめ、他アプリからの流入。
- **インストール体験が標準**  
  タップ → ホーム画面。iPhone ユーザーに一番自然。
- **課金の選択肢が増える**  
  買い切り、アプリ内課金、サブスク（StoreKit）。ただし手数料とルールあり。
- **PWA/ブラウザ版との差別化**  
  「ストア版＝正式・最新・サポートあり」と位置づけやすい。

### 買い手側

- 開くのが簡単（Files 経由が不要）
- OS の管理下で更新・削除が分かりやすい
- 見た目が“ちゃんとしたアプリ”に見える

---

## 5. インストール販売のデメリット

### コスト・手間

- **包む作業が必要**  
  Capacitor / Cordova 等で WebView に載せる、または書き直し。「バンドラなし」方針からは一歩外に出る。
- **Apple Developer 年額（約 $99）**、証明書、審査、リジェクト対応。
- **審査で止まりうる点**（癒し Canvas でも）  
  - ほぼ WebView だけのアプリ（中身が薄いと判断）  
  - 外部決済の誘導  
  - プライバシー文言、トラッキング  
  - クラッシュ・音・横向きなどの動作不備  
- **更新のたびに審査**が基本（緊急でも即配布しにくい）。
- **手数料 15〜30%** がかかる。

### 技術・体験の落とし穴

- Web で動いていたものが、そのままでは落ちることがある（ファイル読み込み、オーディオ、ジャイロ、セーフエリア、背面スワイプなど）。
- ストア規約で「やりたい売り方」が縛られる（例: アプリ内のデジタル機能をストア外決済だけにするのは原則むずかしい）。
- Android も出すなら二倍（テスト・画面・ストア文言）。

## 6. 作品タイプ別の向き・不向き

| 作品 | ダウンロード販売 | インストール販売 |
|------|------------------|------------------|
| **脳リフレクソ** | すでに最適に近い。単体 HTML が武器 | 「正式アプリ版」として単価・信頼アップの余地あり |
| **Fluid Words** | zip/単体化すれば売れる。OCR・CDN 依存はオフライン設計の見直しが必要 | ホーム画面アプリ向き。WebGL の実機検証が重要 |
| **Spacey Bloom** | 単一 HTML なので配布が最も簡単 | 包みやすいが、巨大単体ファイルの運用は工夫が必要 |
| **Nebula Garden** | 4ファイル→単体化すれば脳リフレクソと同型で売れる | 同シリーズの「続編アプリ」として出しやすい |

共通して強い売りは、**短時間・癒し・音と光・考えすぎリセット**。  
「重いRPG」より **衝動買い・ギフト・自分へのケア** の売り方と相性が良い。

---

## 7. 一文で比較

**ダウンロード販売**  
→ 作ったものを早く・安く・自分の条件で売れる。その代わり、iPhoneでの開き方・集客・コピー耐性・“アプリ感”は弱い。

**インストール販売**  
→ 見つけてもらいやすく、高く・信頼して買ってもらいやすい。その代わり、包む工数・年額・審査・手数料・規約がずっと重い。

---

## 8. 実践的なおすすめ（段階）

1. **今すぐ伸ばすなら B（ダウンロード）を磨く**  
   - 単体 HTML 化を全作品で揃える  
   - 「Safariで開く」手順の動画／スクショを短縮  
   - BOOTH 等でシリーズ販売（脳リフレクソ本体＋Nebula＋Fluid など）
2. **売上が安定してきたら C を検討**  
   - まず一番売れている1本だけ Capacitor 等で App Store  
   - Web版はデモ／軽量、ストア版はフル、の二層も可
3. **やらなくてよいこと（初期）**  
   - 全作品のネイティブ書き直し  
   - いきなりサブスク中心（癒しの単発体験は買い切りの方が説明しやすいことが多い）

---

## 9. 決めると迷いが減る論点

- **誰に売るか**: 今の購入者（zipに慣れている層）か、App Store の一般層か
- **価格帯**: zip なら数百〜二千円前後、ストアなら心理的にやや上げやすい、など
- **オフライン必須か**: Fluid の OCR など、ネット前提機能の扱い
- **コピー対策をどこまで気にするか**: 完全防御は諦めて、早く出す／シリーズ化する方が現実的なことが多い
